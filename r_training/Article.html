Entendendo o R e Rstudio

R basicamente uma linguagem de programao e um software estatstico. Foi criado como um upgrade da linguagem S, que foi criada por John Tukey, um dos idealizadores da anlise de dados (formal).

O R funciona basicamente como uma calculadora, onde voc pode inserir operadores matemticos e rodar o cdigo atravs do ctrl + enter. Por exemplo:

10 + 2
## [1] 12

Pode-se tambm criar variveis atravs do comando alt + - que ir criar um <-. Como o R uma linguagem vetorizada, ao criar um vetor e aplicar uma operao nele, o R ir fazer esta operao para todos os elementos do vetor. Um vetor pode ser criado usando c() e inserindo os elementos do vetor dentro dos parenteses.

var_1 <- c(1,2,3,4,5)

# Realizando operaes na varivel / vetor

2*var_1
## [1]  2  4  6  8 10
# Varivel contendo raiz quadrada da varivel 1
var_2 <- sqrt(var_1)

Chamando uma varivel:

var_2
## [1] 1.000000 1.414214 1.732051 2.000000 2.236068

Importante notar, que eu estou no Global Enviroment. Eu prefiro usar este enviroment, pois no uso pacotes que possam entrar em conflito e no realizo projetos muito complexos, desta forma, todas as variveis so criadas no global enviroment e, ao iniciar um novo projeto de pesquisa, eu fao a limpeza do meu enviroment atravs dos comandos:

rm(list = ls())

A funo rm remove os objetos de uma sesso ou enviroment e por ser uma funo, ela precisa de parmetros. Um deles o list, que conter as variveis a serem removidas. Dentro desta lista, eu posso colocar um vetor de variveis para serem removidas. Como eu removo tudo, eu uso a funo ls(), que lista todos os objetos ou variveis do meu enviroment.

Mas como saber que esta funo faz isso? No R, caso voc tenha um problema e no saiba como resolv-lo, a comunidade ser sua melhor parceira. Atualmente, minhas duas fontes para tirar dvidas so:

  • Grupo R Brasil do Telegram (https://t.me/rbrasiloficial)

  • Stack Overflow

Outro cenrio, que exige outro tipo de soluo, quando voc sabe que uma funo pode ser usada para um dado problema, porm no se lembra como us-la. Nesta caso, o prprio R pode ser sua primeira opo. Usando o comando help, o R ir lhe trazer a documentao do pacote ou funo em questo. H duas formas de se pedir ajuda no R.

# Usando interrogao

Apesar de possuir um grande nmero de funes nativas, o principal atrativo desta ferramenta (como o de muitas outras tambm), so os pacotes criados pela comunidade que ficam disponveis para serem usados. Principais Pacotes Usados em Pesquisas

Ao executar projetos de anlise de daods, provavelmente voc precisar utilizar solues que no esto disponveis no base R (funes nativas da ferramenta). Neste caso, voc precisar de pacotes que tenham esta soluo. Os pacotes do R podem ficar armazenados no cran, que significa Comprehensie R Archive Network, ou no github do autor do pacote.

A instalao de pacotes bem simples, basta escrever install.packages(nome do pacote) e rodar. Para ver onde seus pacotes esto sendo armazenados, basta ler o output da instalao ou tambm atravs do comando:

.libPaths()
## [1] "C:/Users/francisco.piccolo/Documents/R/R-3.6.1/library"

Caso queira ver sua lista de pacotes, basta usar o comando:

installed.packages()

Se caso voc tenha um pacote, porm ele esteja desatializado, o comando packageVersion lhe mostrar a verso do pacote e o comando update.packages ir atualizar a verso dele.

packageVersion("ggplot2")

update.packages("ggplot2")

Aps instalar o pacote ou atualiz-lo, necessrio chamar este pacote para sua sesso do R, pois a instalao s faz ele ser inserido naquela pasta indicada em .libPaths(). Trazer o pacote para sua sesso simples, basta usar o comando library() e o nome do pacote, desta vez, no precisa das aspas.

Este mtodo funciona para pacotes j hospedados no Cran. Caso o pacote necessrio esteja no github, necessrio instalar o pacote devtools que conter a funo install_github(), que permitir instalar pacotes do github. Em geral, recomenda-se usar pacotes do Cran, pois semelhante a um selo de qualidade do pacote, onde o mesmo no ir conflitar com outros pacotes existentes e tambm conter a documentao necessria para que a comunidade possa us-lo.

Entendendo Documentos Rmarkdown

Ao fazer anlise de dados ou relatrios, muitas vezes optamos pelo excel ou por apresentaes em ppt, ou at mesmo um e-mail resumindo os insights. No h problema com estas alternativas, porm quando est se desenvolvendo algo mais complexo, onde sero usados grficos, tabelas e tambm partes escritas indicando hipteses, premissas e (o mais importante) concluses, um documento Rmarkdown ou Jupyter Notebook so alternativas interessantes.

No R, para se criar um documento Rmarkdown, basta criar um documento novo usando Ctrl + Shift + N e na hora de salv-lo, basta colocar o formato como .Rmd. Para executar este documento, pressione Ctrl + SHift + K (antes escreva alguma coisa). Veja que o R ir gerar um output na pgina Viewer do Rstudio.

Agora que temos nosso documento HTML sendo gerado, podemos escrever alguns cdigos em R e testar algumas opes nestes code chunks. Nele, voc pode inserir vrios parmetros, que sero avaliados na hora de renderizar (criar o documento HTML) o seu Rmarkdown. Os principais so explicados abaixo:

  • eval (T ou F): executa ou no o cdigo

  • include (T ou F): inclui ou no o output do cdigo

  • warning (T ou F): apresenta ou no a mensagem de aviso do cdigo

  • error (T ou F): apresenta ou no a mensagem de erro do cdigo

  • echo (T ou F): apresenta ou no o cdigo

  • fig.align (center, left ou right): alinha o output do cdigo

  • fig.width e fig.height: altera a largura e altura do output do cdigo

No seu documento Rmarkdown, crie alguns code chunks e teste estes parmetros. Veja que voc pode alterar a linguagem usada no code chunk e o HTML ir utilizar uma syntax highlight especfica para a linguagem indicada. Veja abaixo um exemplo de cdigo SQL:

Conectando o R ao Redshift

Neste tpico, vamos utilizar nosso documento Rmarkdown e alguns pacotes do R para fazer a conexo com o Redshift e extrair data.frames para serem usados nas pesquisas.

Para se conectar ao Redshift, preciso ter o arquivo .jar com o Driver JDBC da Amazon. Este arquivo pode ser baixado neste link. Aps baixar este arquivo, crie um code chunk para instalar e chamar os seguintes pacotes: rJava e RJDBC.

install.packages("rJava")
install.packages("RJDBC")

library(rJava)
library(RJDBC)

Crie uma varivel (i.e.driver) contendo o arquivo .jar baixado. Use o seguinte code chunk para isso:

# Veja que eu indico o pacote que possui a funo JDBC. Isso  interessante, pois permite que outras pessoas, ao lerem meu cdigo, saibam quais pacotes so necessrios
driver <- RJDBC::JDBC(driverClass = "com.amazon.redshift.jdbc41.Driver", 
               classPath = Sys.glob("file_path/file.jar"), 
               identifier.quote="`")

Crie uma varivel (i.e.db_connection) para indicar as informaes do DW que ser conectado pelo R. Use o seguinte code chunk para isso:

db_connection <- sprintf("jdbc:redshift://%s:%s/%s?tcpKeepAlive=true&ssl=true&sslfactory=com.amazon.redshift.ssl.NonValidatingFactory", 
                         "host of the database", 
                         "port", 
                         "database name")

Crie uma varivel (i.e.access) que conter as informaes do DW e tambm as suas credenciais para acessar o DW. Use o seguinte code chunk para isso:

access <- RJDBC::dbConnect(driver, db_connection, "user", "password")

Agora com a conexo ao DW realizada, podemos fazer um teste simples para ver se possvel extrair dados do banco. Para isso, use a funo RJDBC::dbGetQuery() que solicita os valores da varivel access. Este um procedimento prtico de trazer data.frames para o R, porm lembre-se de que o R opera com utilizao da memria RAM, por isso no d para trazer resultados de queries muito grandes, pois no vai caber na memria. Desta forma, preciso realizar a maior parte da manipulao dos dados no SQL e apenas o resultado final para o R. Outro ponto que vale a pena ser lembrado que o R no conseguirar ler uma query de uma tabela temporria, criada no dBeaver (se tem aviso por que j tentaram).

Sabendo que a maior parte da manipualao ser no SQL, fica complicado fazer toda esta operao dentro do R. Pensando nisso, surgiu uma soluo interessante, onde o cdigo SQL armazenado em um arquivo .sql e o R faz a leitura deste arquivo e usa-o como entrada da funo dbgetQuery() vista anteriormente.

Vamos usar um exemplo para ficar mais claro. Crie um arquivo com um cdigo sql e salve-o como .sql. Em seguida, usando o pacote readr, leia o arquivo .sql criado e salve o resultado em uma varivel. Por exemplo:

library(readr)

Veja que eu salvei meu script sql na varivel sql_script.

Vamos testar essa soluo com um exemplo:

No arquivo usado anteriormente, adicione uma nova query. Para que esta soluo funcione, necessrio separar cada query por um texto que indique o comeo e fim desta query. Desta forma, seu arquivo .sql ficar:

Agora podemos realizar diversas consultas ao DW e trazer os resultados para o Rstudio para realizar as anlises. Isso quase tudo o que voc precisar para executar seus projetos e pesquisas. O ltimo ponto que falta o armazenamento do output do SQL em um arquivo compactado, de forma que voc no precise fazer a consulta ao DW toda vez que quiser renderizar seu documento. O renderizar quando se executa seu Rmarkdown atravs do Ctrl + Shift + K.

Imagine que voc faa uma alterao simples em um grfico ou corrija uma palavra e precise renderizar seu documento, porm voc executa queries um pouco pesadas. Isso iria criar um gargalo e afetaria a produtividade do seu trabalho.

Para solucionar isso, atualmente eu adoto uma metodologia que funciona da seguinte forma. Ao criar uma varivel com o output da minha query, por exemplo query_1, eu salvo este output em uma pasta (recomendo ser na mesma pasta em que estou realizando a pesquisa). Logo abaixo deste cdigo que salva o output, eu fao um novo cdigo que ir ler este arquivo e este cdigo que ir rodar toda vez que eu renderizar meu documento Rmarkdown.

Para facilitar o entendimento, vamos ver um exemplo:

# Criando uma varivel contendo o output de uma query
query_1 <- RJDBC::dbGetQuery(access,
                  substr(sql_script,
                         min(stringr::str_locate(string = sql_script, "-- query_2_begin")),
                         max(stringr::str_locate(string = sql_script, "-- query_2_end"))))

Agora vou criar o arquivo .RDS (arquivo que lido pelo R). No code chunk desta etapa e da anterior, eu indico eval = F, ou seja, o cdigo no ser executado na renderizao do documento.

# Usando a funo write_rds para gravar o output do SQL em uma pasta
readr::write_rds(query_1, "C:/Users/francisco.piccolo/Desktop/R/01.DFT Research/r_training/datasets/query_1.RDS")

Por fim, em um novo code chunk, eu insiro a funo read_rds e salvo o resultado na mesma varivel query_1 criada anteriormente.

# Salvando o resultado em uma varivel
readr::read_rds("C:/Users/francisco.piccolo/Desktop/R/01.DFT Research/r_training/datasets/query_1.RDS") -> query_1

Veja que eu atribuo o resultado a uma varivel usando o ->. uma alternativa para se criar variveis.

Com isso, eu preciso apenas rodar o SQL durante o desenvolvimento do estudo, na parte mais inicial do projeto. No final do projeto, o foco mais em ajustes de detalhes, no precisando rodar novamente toda a query.

Caso seu estudo precise rodar com frequncia, para pegar dados recentes, a alternativa no ser interessante, pois ser necessrio rodar o cdigo SQL a cada renderizao do documento.

Entendendo os mtodos de criao de documentos Rmarkdown, conexo ao DW e gerao de arquivos RDS para salvar os resultados da consulta ao DW, podemos seguir para a parte que cria valor para o negcio, o desenvolvimento de pesquisas.

Desenvolver pesquisas no R fica bem simples com a utilizao do Rmarkdown. Os componentes bsicos de uma pesquisa so:

  2. Premissas

  3. Anlise de Dados

Este tpico, ir focar no item 3, onde iremos utilizar alguns dados do DW para gerar visualizaes e chegarmos em concluses e insights sobre o negcio. Para fazer esta atividade, vamos usar o pacote tidyverse que contm vrios outros pacotes usados em anlise de dados.

Antes de iniciar a programar, vamos refletir um pouco sobre o desenvolvimento de pesquisa. Quando eu vou desenvolver uma pesquisa, busco abordar um tpico bem especfico e esta especificao eu fao atravs de uma pergunta, que ser o ponto de partida para formulao da minha hiptese. Veja alguns exemplos de pesquisas realizadas recentemente:

Note que o ttulo e o subttulo j indicam algo bem especfico que ser estudado. Com isso, voc consegue aprofundar o tema, sem se perder na gerao de concluses. Esta uma prtica que eu venho adotando e tem funcionado.

Pode ocorrer de voc desenvolver uma pesquisa e perceber que seu tpico ou sua pergunta est muito aberta.
